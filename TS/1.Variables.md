# Variables en JavaScript/TypeScript

JavaScript es dinámicamente tipado, pero con TypeScript puedes agregar tipado estático opcional para mayor seguridad.

## Declaración de Variables

| Palabra clave | Uso recomendado | Alcance |
|--------------|-----------------|---------|
| `const` | ✅ Valores que no cambian | Bloque |
| `let` | ✅ Valores que cambian | Bloque |
| `var` | ❌ Evitar - legacy | Función |

```typescript
const PI: number = 3.14159;        // ✅ Constante - no cambia
let contador: number = 0;          // ✅ Variable mutable
// var -> ❌ No usar (problemas de scope y hoisting)
```

## Tipos de Datos

### Primitivos
```typescript
const edad: number = 19;                    // Enteros y decimales
const saldo: number = 12.54;
const nombre: string = 'Juan';              // Comillas simples (convención)
const mensaje: string = `Hola ${nombre}`;   // Backticks para interpolación
const activo: boolean = true;
const vacio: null = null;                   // Intencionalmente vacío
const sinAsignar: undefined = undefined;    // Valor no inicializado
const idUnico: symbol = Symbol('id');       // Identificador único (ES6+)

// BigInt para números muy grandes (ES2020+)
const numeroGrande: bigint = 9007199254740991n;
```

### Estructuras de Datos

```typescript
// Arrays tipados
const numeros: number[] = [1, 2, 3];
const nombres: Array<string> = ['Ana', 'Luis'];  // Notación genérica alternativa

// Tuplas (arrays de longitud fija con tipos específicos)
const coordenadas: [number, number] = [10, 20];
const usuario: [string, number] = ['Juan', 25];

// Enums para conjuntos de constantes
enum Color {
  Rojo = 'rojo',
  Verde = 'verde',
  Azul = 'azul'
}
const miColor: Color = Color.Rojo;
```

### Objetos
```typescript
// Interface (recomendado para objetos)
interface Producto {
  readonly id: number;      // Solo lectura
  name: string;
  color?: string;           // Opcional
  size: number;
  enStock: boolean;
}

const objeto: Producto = {
  id: 1,
  name: 'ball',
  color: 'black',
  size: 12,
  enStock: true
};

// Type alias (alternativa a interface)
type Punto = {
  x: number;
  y: number;
};

// Clases
class Persona {
  constructor(
    public nombre: string,
    private edad: number    // Acceso restringido
  ) {}
  
  saludar(): string {
    return `Hola, soy ${this.nombre}`;
  }
}
```

### Tipos Especiales
```typescript
// NaN detection
const resultado: number = NaN;
const esInvalido: boolean = Number.isNaN(resultado);  // ✅ Método específico

// Unknown (más seguro que any)
let valorDesconocido: unknown = 4;
valorDesconocido = 'texto';

// Void (funciones sin retorno)
function logMensaje(msg: string): void {
  console.log(msg);
}

// Never (funciones que nunca retornan)
function lanzarError(mensaje: string): never {
  throw new Error(mensaje);
}
```

## Template Literals (Backticks)
```typescript
const amigo: string = 'KD';
const numero: number = 2;
const saludo: string = `Bienvenido, ${amigo}, ${numero * 3} veces`;

// Tagged templates (avanzado)
function etiquetar(strings: TemplateStringsArray, ...values: number[]): string {
  return strings.reduce((acc, str, i) => acc + str + (values[i] || ''), '');
}
```

## Type Assertions y Guards
```typescript
// Type guard
function procesar(valor: string | number): void {
  if (typeof valor === 'string') {
    valor.toUpperCase();  // TS sabe que es string aquí
  } else {
    valor.toFixed(2);     // TS sabe que es number aquí
  }
}
```

## 🗑️ Lo que se ELIMINÓ (ya no usar)

1. **`var`** → Usar `const`/`let` (evita hoisting problemático y scope global)
2. **Comillas dobles para strings** → Preferir simples o backticks (consistencia)
3. **Arrays sin tipo/mezclados** → Tipar arrays en TS (`number[]`, `string[]`)
4. **`document.write()`** → Peligroso, usa DOM manipulation (`textContent`, `innerHTML` con cuidado)
5. **`prompt()`/`alert()`** → UX obsoleta, bloqueantes; usar formularios/modales
6. **Concatenación con `+`** → Usar template literals `` `${}` `` (más legible)
7. **`isNaN()` global** → Usar `Number.isNaN()` (no hace coerción de tipos)
8. **Variables sin declarar** → Siempre declarar con `const`/`let`
9. **Objects sin interface/type** → Definir estructura en TypeScript
10. **Uso de `undefined` explícito** → Preferir `null` para valores intencionalmente vacíos
11. **`parseInt()` sin radix** → `parseInt(x, 10)` o `Number()` (Comportamiento inconsistente)
12. **`any` implícito** → Tipos explícitos o `unknown` (Desactiva verificación)
13. **Funciones con `function`** → Arrow functions `=>` (Lexical `this`, más conciso)
14. **`==` (doble igual)** → `===` (triple igual) (Comparación estricta)
